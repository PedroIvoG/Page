Visão geral

Projeto de dashboard executivo de vendas desenvolvido no Power BI com integração direta a uma base SQL Server. O objetivo foi centralizar dados de múltiplas filiais em um único painel analítico, permitindo acompanhamento de metas em tempo real e análise em diferentes granularidades — do nível nacional até o SKU individual.

O problema

A empresa não possuía visibilidade centralizada de suas vendas. Os dados estavam fragmentados em planilhas Excel por filial — cada gerente regional mantinha sua própria versão, tornando impossível uma visão consolidada e confiável.

O resultado: reuniões de resultado atrasadas, metas acompanhadas de forma manual e decisões tomadas com dados defasados de semanas.

Solução desenvolvida

Construí um pipeline de dados ponta a ponta — da extração no SQL Server até a publicação no serviço Power BI — com atualização automática diária. O painel entregou:

  • Acompanhamento de meta mensal com gauge dinâmico e indicadores de variação vs período anterior
  • Análise de sazonalidade com drill-down por mês, trimestre e ano fiscal
  • Mapa de calor por região com drill-through para nível de filial e vendedor
  • Ranking de produtos por receita, ticket médio e margem de contribuição
  • Filtros dinâmicos com slicers sincronizados e bookmarks de visões pré-salvas
  • RLS (Row Level Security) para cada gestor regional ver apenas sua área

Medida DAX principal

Cálculo da variação percentual em relação à meta acumulada no ano:

DAX
-- Variação % realizado vs meta (Year-to-Date) Var_Meta_YTD = VAR VendasYTD = CALCULATE( SUM( Vendas[Valor] ), DATESYTD( Calendario[Data] ) ) VAR MetaYTD = CALCULATE( SUM( Metas[ValorMeta] ), DATESYTD( Calendario[Data] ) ) RETURN DIVIDE( VendasYTD - MetaYTD, MetaYTD, 0 )
